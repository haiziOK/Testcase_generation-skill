# 输出要求

## 所有Excel文件

### 专业格式
- 对所有交付物使用一致的专业字体（如Arial、Calibri）
- 应用适当的单元格格式以提高可读性（自动换行、合适的列宽）
- 包含带有对比背景颜色的清晰标题
- 使用颜色编码表示测试优先级（红色-高、黄色-中、绿色-低）

### 测试用例结构
- 每个测试用例必须包含：ID、标题、描述、步骤、预期结果、优先级、需求ID
- 步骤应编号清晰
- 预期结果应具体且可验证
- 优先级应根据风险和重要性分配

### 零公式错误
- 每个Excel文件必须无公式错误（#REF!、#DIV/0!、#VALUE!、#N/A、#NAME?）
- 如果未来版本添加公式，请适当使用单元格引用

## 测试用例生成

### 覆盖要求
- 生成涵盖正向、负向和边界场景的全面测试用例
- 确保测试用例可追溯至需求
- 基于风险评估确定测试用例优先级
- 在适用时包含功能性和非功能性测试用例

### 输入处理
- 支持多种文档格式：TXT、PDF、DOCX
- 支持图谱格式：JSON、GraphML（未来版本）
- 通过高效处理处理大型文档
- 为不支持的格式或解析失败提供清晰的错误消息

## 技能使用

### 何时使用此技能
在以下情况使用此技能：
- 您有设计文档并需要生成测试用例
- 您想创建结构化的Excel测试用例存储库
- 您需要将现有测试文档转换为标准化格式
- 您想利用现有的测试用例图谱和经验总结

### 何时不使用此技能
在以下情况不使用此技能：
- 主要交付物不是Excel文件（使用其他适当的技能）
- 您需要执行测试（使用测试工具代替）
- 您只需要分析现有测试用例而不生成新的

## 交互式提示

当调用时，技能将引导用户完成：
1. **设计文档选择**：选择或提供设计文档路径
2. **图谱输入（可选）**：可选提供测试用例图谱或经验图谱
3. **输出配置**：指定输出Excel文件路径和生成选项
4. **生成选项**：选择测试用例粒度、覆盖级别和优先级分布

当用户未提供特定输入时，技能使用智能默认值。

## 实现说明

### 核心依赖
- **文档处理**: PyPDF2、pdfplumber、python-docx 用于解析设计文档
- **LLM集成**: 使用本地Ollama或OpenAI进行测试用例生成
- **Excel生成**: openpyxl 用于创建格式化的Excel文件
- **图谱处理**: NetworkX 用于图谱操作（未来版本）

### 错误处理
- 当LLM服务不可用时优雅降级
- 为不支持的格式提供清晰的错误消息
- 交付前验证输出Excel文件
- 记录生成过程以便调试

### 性能考虑
- 通过分块高效处理大型文档
- 缓存已解析文档以供重复生成
- 可配置的LLM请求超时
- 大型测试用例集的内存管理

## 最佳实践

### 对于生成的测试用例
1. **清晰性**: 每个测试步骤应清晰且可操作
2. **独立性**: 测试用例应尽可能独立
3. **可追溯性**: 保持与源需求的清晰链接
4. **可维护性**: 构建易于更新的测试用例结构

### 对于Excel格式
1. **一致性**: 在所有工作表中使用一致的格式
2. **可读性**: 确保文本可读，具有适当的列宽
3. **组织性**: 将相关的测试用例分组
4. **导航性**: 考虑为大型测试套件添加目录

## 未来增强

计划在未来版本中添加的功能：
1. **图谱集成**: 完全支持JSON/GraphML测试用例图谱
2. **经验学习**: 融入测试经验总结以进行更智能的生成
3. **模板支持**: 不同测试方法学的自定义Excel模板
4. **API暴露**: 用于CI/CD管道集成的REST API
5. **协作功能**: 多用户编辑和版本控制

此技能为自动化的测试用例生成提供了基础，可以根据用户需求和反馈进行扩展。
